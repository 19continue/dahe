# 大禾种业微信小程序项目简历写法

## 1. 项目名称怎么取

推荐按“业务场景 + 系统属性”命名，不建议直接把仓库名写进简历。

### 版本 A：最稳

农业种业业务管理系统

### 版本 B：更偏小程序

农业种业微信小程序与后台管理系统

### 版本 C：更偏业务价值

农业现场数据采集与质量追溯系统

如果你主要投 Java 后端，优先用版本 A 或版本 C，看起来更像完整业务系统，不会把自己限定成“只会写小程序页面”。

## 2. 技术栈怎么写最安全

### 2.1 推荐写法

`uni-app`、`Vue`、`tdesign-uniapp`、`Vue3`、`Vite`、`Element Plus`、`Spring Boot`、`Spring Security`、`MyBatis-Plus`、`MySQL`、`Redis`、`JdbcTemplate`、`高德地图开放平台`

### 2.2 如果你想更保守

`uni-app`、`Vue`、`Spring Boot`、`MyBatis-Plus`、`MySQL`、`Redis`

### 2.3 不建议写的技术词

1. `Spring Cloud`
2. `Kafka`
3. `RabbitMQ`
4. `分库分表`
5. `分布式锁`
6. `JWT 网关`
7. `高并发秒杀`

这些词当前项目里没有真实支撑，写上去风险很大。

## 3. 项目描述怎么写

### 3.1 100 字版本

该项目是一个面向农业种业场景的业务管理系统，包含微信小程序端、后台管理端和 Spring Boot 后端，围绕用户认证、田块分布、农事记录、种子批次、质量检测、消息通知等业务展开。后端通过 `/api/v2/miniapp/**` 与 `/api/v2/admin/**` 分离小程序与后台能力，并结合 MySQL、Redis、文件上传和高德位置天气能力支撑业务落地。

### 3.2 150 字版本

该项目是一个服务农业种业场景的业务管理系统，采用前后端分离实现，包含微信小程序端、后台管理端和 Spring Boot 后端。系统围绕用户认证准入、田块管理、种植计划、农事记录、种子批次、质量检测、企业介绍、消息通知等模块展开，并接入高德地图天气、文件上传、动态表单配置和资源审核能力。后端通过统一的 miniapp/admin 路由边界实现权限隔离，Redis 主要用于会话读取加速和高德接口缓存。

### 3.3 面向后端岗位的增强版

该项目是一个农业种业业务管理系统，包含小程序端、后台端和 Spring Boot 后端三部分。后端采用模块化单体架构，围绕用户认证、田块与种植计划、农事记录、种子批次与检测、资源上传审核、消息通知和地图天气能力展开。项目中实现了小程序与后台路由隔离、token session + Redis 会话缓存、动态表单 schema 治理、种子检测幂等写入、资源引用追踪与异步消息派发等能力。

## 4. 项目经历可直接写法

下面这段是可直接放简历的版本，你可以按掌握程度删减。

### 4.1 安全版

项目名称：农业种业业务管理系统  
项目描述：该项目包含微信小程序端、后台管理端和 Spring Boot 后端，主要服务于农业种业场景下的用户认证、田块管理、农事记录、种子批次与质量检测等业务。

负责内容：

1. 负责小程序与后端联调，完成认证登录、田块分布、农事记录、种子批次与检测等核心业务功能开发。
2. 梳理小程序用户准入流程，支持未申请、审核中、未通过、已通过、已停用、资格已收回等状态在登录页的差异化处理。
3. 参与田块与农事记录链路开发，支持流程步骤驱动的农事录入、动态参数存储和时间轴展示。
4. 参与种子批次与检测模块开发，支持检测记录新增、编辑、样本数计算和幂等写入处理。
5. 接入高德位置与天气能力，支持附近田块展示、当前位置匹配和首页天气信息展示。

### 4.2 亮点版

项目名称：农业现场数据采集与质量追溯系统  
项目描述：系统包含微信小程序端、后台管理端和 Spring Boot 后端，围绕田块、农事、种子批次、检测记录、资源图片和消息通知构建完整业务链路。

负责内容：

1. 负责小程序认证准入链路开发，复用后端认证能力实现“先认证、后登录”的小程序账号体系，并完成多状态登录页交互设计。
2. 负责田块与农事记录模块开发，支持根据当前位置匹配当前田块、按流程步骤录入农事数据，并通过动态表单 schema 扩展步骤字段。
3. 负责种子检测模块开发，增加 `requestKey` 幂等机制，避免重复提交通路下产生重复检测记录。
4. 参与后端 Redis 能力接入，将其用于 token session 读取加速和高德位置天气缓存，降低数据库与第三方接口压力。
5. 参与消息通知与资源上传链路开发，支持异步派发消息任务以及小程序图片上传后的审核治理。

## 5. 可以挑选的亮点条目

下面这些条目你不用全写，选 4 到 6 条即可。

1. 基于小程序认证状态实现“先认证、后登录”的准入机制，支持审核中、未通过、已停用、资格已收回等多状态处理。
2. 采用 `/api/v2/miniapp/**` 与 `/api/v2/admin/**` 分离小程序和后台接口，降低端侧混用和权限串线风险。
3. 使用 Redis 对 token session 做读取加速缓存，并设计 Redis 故障短暂 bypass 回退数据库的处理逻辑。
4. 接入高德地图位置与天气能力，支持附近田块展示、当前位置匹配和首页天气快照，并通过本地缓存 + Redis 缓存降低真实接口调用。
5. 实现流程步骤驱动的农事记录录入，支持动态表单 schema 扩展字段并以 `extraJson` 存储业务参数。
6. 在种子检测模块增加 `requestKey` 幂等机制，避免重复提交时生成重复检测记录。
7. 实现资源上传、审核、回收和业务引用追踪，支持小程序图片与用户头像等资源的统一治理。
8. 通过 `@Async + ThreadPoolTaskExecutor` 实现消息任务异步派发，避免大批量通知阻塞主业务请求。
9. 实现田块和种子批次的搜索建议能力，支持关键词、拼音前缀和模糊联想。

## 6. 简历里哪些写法最容易翻车

### 6.1 不要这样写

1. 精通高并发分布式架构设计
2. 负责微服务拆分与网关治理
3. 基于 Kafka 实现消息最终一致性
4. 基于 JWT + Gateway 完成统一认证授权
5. 通过分库分表解决海量数据问题

### 6.2 原因

1. 当前项目后端不是微服务。
2. 当前项目没有 Kafka、MQ、网关、分库分表这些实现。
3. 当前项目的认证更接近“token session + 自定义鉴权拦截”。
4. 如果写成大厂式分布式项目，面试官会立刻往更深处追问。

## 7. 针对你现在的最优写法建议

如果你当前最主要目的是春招找 Java 后端岗位，我建议你最后落成下面这一版：

项目名称：农业种业业务管理系统  
技术栈：`uni-app`、`Vue`、`Vue3`、`Spring Boot`、`Spring Security`、`MyBatis-Plus`、`MySQL`、`Redis`、`Element Plus`、`高德地图开放平台`

项目描述：
该项目包含微信小程序端、后台管理端和 Spring Boot 后端，围绕用户认证、田块分布、农事记录、种子批次、质量检测、资源上传和消息通知等农业业务展开。后端通过 miniapp/admin 路由隔离不同端侧能力，并结合 Redis 会话缓存、高德位置天气缓存、动态表单配置和异步消息派发支撑系统运行。

负责内容：

1. 负责小程序认证与登录链路开发，支持用户申请、审核状态识别、微信登录及登录态维护。
2. 负责田块与农事记录相关功能开发，支持当前位置匹配田块、流程步骤录入和动态参数存储。
3. 负责种子批次与检测模块部分后端能力开发，补齐检测记录幂等写入和发芽率计算逻辑。
4. 参与 Redis 会话缓存、高德位置天气缓存和图片上传审核链路实现。
5. 参与后台消息通知与个人收件箱能力开发，支持消息异步派发和已读删除。

这版的优点是：

1. 不虚。
2. 亮点够用。
3. 面试时比较容易接住。

## 8. 你可以怎么继续微调

1. 如果你对后台管理端很熟，可以保留“后台管理端”表述。
2. 如果你主要熟后端，就把前端词汇压缩，重点写后端设计和业务链路。
3. 如果你对某个亮点没把握，就删掉，不要为了好看硬留。
