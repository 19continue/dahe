# DaHeServeV2 `assets` 模块待改进的点（最终版）

- 更新时间：2026-03-11
- 对应解读文档：`docs/backend-module-review/modules/assets-模块解读.md`

## 已完成

1. 后台资源治理从单纯“资源列表操作”提升为独立治理服务。
- 已新增 `MediaAssetAdminService`，把资源锁、驳回入回收站、删除前解锁等后台治理动作从控制器中下沉出来。

2. 增加资源锁机制。
- 已支持超级管理员设置、关闭、重置资源锁。
- 已支持普通管理员删除锁定资源时输入解锁密码。
- 密码已改为摘要存储。

3. 审核增加“驳回”。
- 驳回时必须填写原因。
- 驳回后自动进入回收站，避免误审直接丢失资源。

4. 后台资源管理从单页混合治理改为分页面治理。
- 已拆出资源总览、资源库、回收站、目录治理三类页面。

5. 资源列表支持锁定筛选。
- 后端分页已新增 `lockedFlag` 过滤条件。

## P1

1. 控制器继续拆分。
- 现状：`MediaAssetController` 仍承载上传、资源治理、目录治理、统计与部分公共读取能力。
- 建议：至少拆成“上传入口 / 资源治理 / 目录治理 / 统计入口”四类控制器。

2. 目录与统计 SQL 继续下沉。
- 现状：目录治理与统计逻辑仍存在直接 `JdbcTemplate` 查询。
- 建议：补齐 repository 或独立 query service，避免控制器与底层 SQL 再次耦合。

## P2

1. 小程序公共运营资源需要单独映射层。
- 现状：`assets` 已具备底库与治理能力，但不适合直接承担启动页、登录页、分享图等小程序公共资源编排。
- 建议：后续新增“小程序公共资源映射表 + manifest 接口”，将公共页面资源与业务附件彻底分离。

2. 存储层抽象。
- 现状：当前仍以本地文件系统为主。
- 建议：为 OSS / S3 等对象存储预留适配层。

3. 自动化测试补齐。
- 重点补：
  - 资源锁密码校验
  - 驳回入回收站
  - 严格来源彻底删除保留期
  - 目录竞态删除
