# DaHeServeV2 `auth` 模块待改进的点（最终版）
- 更新日期：2026-03-05
- 对应文档：`docs/backend-module-review/modules/auth-模块解读.md`

## 当前已收口项（仅保留结论）
1. 菜单权限码改为路由自动发现，不再维护硬编码菜单目录。
2. `@AdminMenuCode` 已生效并参与授权与菜单码返回。
3. 角色管理控制层业务已下沉到 `AdminRoleManageService`。
4. 后台认证控制层会话归一化逻辑已下沉到 `AuthPortalService`。

## P0（最高优先级）
1. 共享路由继续收敛
- 目标：将历史 `/api/v2/**` 共享写接口逐步迁移到 `admin/miniapp` 前缀。
- 原因：共享规则越多，鉴权复杂度与误配概率越高。

2. 新增路由鉴权漏配检测
- 目标：对新增写接口做自动检测，避免漏鉴权。
- 方案：启动扫描 + 单测断言双保险。

3. 超级管理员关键操作审计增强
- 目标：对“禁用/删除/解绑超管角色”等尝试打高优先级审计日志。

## P1（近期）
1. 共享规则配置化
- 目标：将 `AuthApiRouteAuthorizationService` 的共享规则从代码迁到配置/表驱动。

2. 会话失效策略细化
- 目标：角色变更后支持“按用户精准失效会话”，减少无关用户受影响。

3. 密码生命周期治理
- 目标：补齐首次改密、周期失效、后台重置流程。

## P2（中期）
1. 鉴权观测面板
- 指标：拒绝原因分布、接口拒绝率、端侧分布、超管操作轨迹。

2. 权限元数据治理
- 目标：路由权限目录、前端菜单键、联调状态形成统一元数据基线。

## 固定规则（防回退）
1. 后台只走角色-菜单路径码模型，禁止回退到写死角色名。
2. 小程序不参与角色体系，只走 `approved + canConsole`。
3. 业务模块不写散落鉴权判断，统一走 auth 模块。
4. 新增模块必须先定路由边界，再实现业务。
