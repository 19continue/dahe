# DaHeServeV2 `crop` 待改进的点（总版）
- 更新时间：2026-03-05
- 对应解读文档：`docs/backend-module-review/modules/crop-模块解读.md`

## 模块目标
保障作物分类/品种链路在并发写入、跨表同步、删除边界上的一致性与可维护性。

## 本轮已完成（P0）
1. 控制层业务下沉（已完成）
- `CropController` 改为协议层，业务编排统一下沉 `CropAdminFacadeServiceImpl`。

2. 跨表同步事务化（已完成）
- 分类/品种改名同步 `field`、`seed_batch` 不再吞异常。
- 同步失败会回滚当前写链路，避免“主表成功、关联表失败”。

3. 删除前引用保护（已完成）
- 删除分类/品种前增加 `farm_process_template.crop_id` 引用检查。
- 被引用时返回明确业务提示，拒绝删除。

4. 唯一性约束落地（已完成）
- 新增 `uk_crop_category_name`、`uk_crop_variety_parent`。
- 启动自举增加历史去重，再建唯一索引。

## 本轮已完成（P1）
1. 新鉴权路径收口（已完成）
- `@AdminMenuCode("/crop-manage")` 对齐后台菜单权限码。
- `/api/v2/crops/**` 在共享路由授权中明确为“后台可访问、小程序拒绝”。

## 仍待处理（P1）
1. 分页查询默认类型语义
- 当前未传 `nodeType` 时默认按 `variety` 查询。
- 建议后续改为显式必传，避免隐式行为。

2. 分类/品种状态化管理
- 当前仅逻辑删除，无 `enabled` 开关。
- 建议后续补充启停能力，支持“下线但不删除”。

## 可排期优化（P2）
1. 作物树短缓存与失效策略。
2. 补齐并发写入与重排冲突场景测试。
3. 改名同步增加操作日志明细（变更前后值、影响行数）。
