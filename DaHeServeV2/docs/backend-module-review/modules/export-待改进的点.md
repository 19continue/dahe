# DaHeServeV2 `export` 待改进的点（最终版）
- 更新时间：2026-03-05
- 对应解读文档：`docs/backend-module-review/modules/export-模块解读.md`

## P0（高优先级）
1. 导出大数据量仍为内存构建
- 现状：当前导出是“查全量 -> 内存拼装 -> 一次性写出”。
- 风险：在数据规模持续增长时，可能出现内存峰值高和响应变慢。
- 建议：升级为分页流式写出，或异步离线导出任务。

2. 导出行为缺少审计指标
- 现状：已有操作日志，但缺少导出耗时/行数/模板命中率等指标。
- 风险：性能问题难以提前发现和量化评估。
- 建议：增加导出埋点并接入统计看板。

## P1（中优先级）
1. 模板发布策略可进一步明确
- 现状：按 `enabled + version_no desc` 取模板，未强制“同模板仅一个启用版本”。
- 风险：运营配置不规范时会出现语义歧义。
- 建议：补充“发布/下线”流程，增加启用态约束。

2. 字段词典与默认列头双源维护
- 现状：词典缺失时仍会回退到代码内置列头。
- 风险：长周期后，词典配置与代码回退规则可能漂移。
- 建议：以词典为主源，回退仅做启动兜底并输出告警日志。

## P2（低优先级）
1. 导出列级脱敏能力
- 现状：尚未提供按字段/角色的脱敏策略。
- 建议：在导出模板层增加脱敏规则配置，满足后续合规要求。

2. 自动化测试覆盖仍可补强
- 现状：缺少导出链路的专项回归用例（模板回退、动态字段冲突、边界行数）。
- 建议：补齐 service 层与 controller 层集成测试矩阵。
