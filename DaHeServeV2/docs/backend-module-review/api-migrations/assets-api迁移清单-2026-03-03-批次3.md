# assets API 迁移清单（批次3最终版）

- 更新时间：2026-03-11
- 范围：DaHeServeV2、DaHeAdminV2、DaHeAppV2
- 说明：本清单记录 `assets` 模块自“小程序接口迁移”以来，到“资源锁 + 驳回入回收站 + 后台拆页”这一轮为止的最终对接口径。

## 1. 后端 API 当前口径（DaHeServeV2）

### 1.1 小程序接口

1. `GET /api/v2/miniapp/assets`
- 小程序资源分页读取。
- 严格来源仅返回可见资源。

2. `POST /api/v2/miniapp/files/upload`
- 小程序资源上传。
- 统一走上传门面服务。

### 1.2 后台接口

1. `POST /api/v2/files/upload`
- 后台资源上传。
- 当前已收口为后台专用。

2. `GET /api/v2/admin/assets`
- 后台资源分页。
- 本轮新增筛选参数：`lockedFlag`。

3. `DELETE /api/v2/admin/assets/{id}`
- 资源移入回收站。
- 本轮新增可选请求体：
```json
{
  "unlockPassword": "******"
}
```
- 用于普通管理员删除锁定资源时输入解锁密码。

4. `POST /api/v2/admin/assets/{id}/restore`
- 回收站恢复资源。

5. `DELETE /api/v2/admin/assets/{id}/purge`
- 彻底删除资源。
- 本轮同样支持可选请求体 `unlockPassword`。

6. `PUT /api/v2/admin/assets/{id}/review`
- 审核资源。
- 当前支持：`pending / approved / rejected`。
- 本轮新增行为：`rejected` 时资源自动进入回收站。

7. `PUT /api/v2/admin/assets/{id}/lock`
- 本轮新增。
- 用于设置、关闭、重置资源锁。
- 请求体示例：
```json
{
  "locked": 1,
  "password": "123456",
  "lockRemark": "品牌主视觉，删除前请先确认替换资源"
}
```

## 2. Admin 前端迁移（DaHeAdminV2）

### 2.1 资源管理页面结构

原结构：
- 单一 `/assets` 页面混合承载资源库、回收站、目录治理。

现结构：
1. `/assets`
- 资源总览页。

2. `/assets/library`
- 资源库页。

3. `/assets/recycle`
- 回收站页。

4. `/assets/folders`
- 目录治理页。

说明：
- 三个子页面仍归属 `/assets` 这一组后台资源管理路由体系。
- 页面拆开后，资源治理职责更清晰，但未引入新的固定角色判断。

### 2.2 Admin 接口适配

1. `removeAsset(id, payload)`
- 已支持给删除接口附带 `unlockPassword`。

2. `purgeAsset(id, payload)`
- 已支持给彻底删除接口附带 `unlockPassword`。

3. `updateAssetLock(id, payload)`
- 已新增对接 `PUT /api/v2/admin/assets/{id}/lock`。

### 2.3 资源治理交互变化

1. 锁定资源删除
- 普通管理员删除时需要输入密码。
- 超级管理员可直接删除或重置锁。

2. 审核驳回
- 后台审核弹窗已明确提示：驳回后资源将自动进入回收站。

3. 资源筛选
- 资源库与回收站均已支持按 `lockedFlag` 筛选。

## 3. 小程序前端状态（DaHeAppV2）

1. 小程序侧无需新增对接本轮后台治理接口。
2. `assets` 小程序请求仍维持：
- `GET /api/v2/miniapp/assets`
- `POST /api/v2/miniapp/files/upload`
3. 本轮资源锁、驳回入回收站、后台拆页均为后台治理能力增强，不改变小程序接口路径。
