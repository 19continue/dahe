# Auth API 迁移清单（2026-03-11，批次7）
- 影响模块：`auth`
- 对接范围：`DaHeAdminV2`
- 说明：本批次主要处理后台消息通知、小程序用户回收、并发冲突保护。

## 1. 新增接口
### 1.1 消息通知管理
1. `GET /api/v2/admin/messages`
- 用途：消息任务分页查询。
- 关键参数：`keyword`、`noticeType`、`targetType`、`dispatchStatus`、`page`、`pageSize`。

2. `POST /api/v2/admin/messages`
- 用途：创建消息任务。
- 关键字段：
  - `title`
  - `content`
  - `noticeType`
  - `targetType`
  - `routeCode`
  - `targetUserIds`
- 行为：创建后异步派发，不阻塞当前请求。

3. `DELETE /api/v2/admin/messages/{id}`
- 用途：删除消息任务。
- 请求体：`expectedUpdatedAt`
- 行为：同步软删除已投递的 `user_notice` 记录。

### 1.2 小程序用户回收治理
1. `POST /api/v2/admin/users/{id}/recycle`
- 用途：将已通过的小程序用户移入回收站。
- 请求体：
  - `expectedUpdatedAt`
  - `recycleRemark`

2. `POST /api/v2/admin/users/{id}/restore`
- 用途：从回收站恢复小程序用户。
- 请求体：
  - `expectedUpdatedAt`

### 1.3 后台通知中心
1. `PUT /api/v2/admin/auth/me/notices/read-all`
- 用途：当前后台用户全部已读。

## 2. 变更接口
### 2.1 小程序用户治理相关接口增加并发保护
以下接口现在都要求前端在请求体中提交 `expectedUpdatedAt`：
1. `PUT /api/v2/admin/users/{id}/approve`
2. `PUT /api/v2/admin/users/{id}/role`
3. `PUT /api/v2/admin/users/{id}/enabled`
4. `DELETE /api/v2/admin/users/{id}`

## 4. 注意事项
1. `targetType=admin_route` 时，后端会按后台路由权限过滤收件人。
2. `DELETE /api/v2/admin/messages/{id}` 与审核类接口一样，存在并发冲突保护，不应省略 `expectedUpdatedAt`。
3. 已通过的小程序用户必须先回收，再允许彻底删除。
