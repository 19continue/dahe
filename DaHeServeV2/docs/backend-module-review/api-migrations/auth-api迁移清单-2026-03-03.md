## 2. 当前鉴权实现（无新增路由）

1. 鉴权入口统一
- 统一走：`ApiAuthInterceptor + AuthApiRouteAuthorizationService`。
- controller 层不再承担路由权限判断。

2. 后台权限
- 后台接口按角色菜单权限控制。
- 对未配置权限映射的后台路由，默认拒绝。

3. 小程序权限
- 小程序不走角色模型，仅使用 `approved + canConsole`。
- 后台与小程序路由前缀严格隔离。

## 3. 本次行为更新（2026-03-04）

1. `POST /api/v2/files/upload`
- 变更前：允许匿名上传（受限场景）。
- 变更后：必须登录后上传（统一纳入鉴权拦截链）。
- 权限规则：
  - 后台用户需具备 `assets` 菜单权限。
  - 小程序用户可访问（通过会话与状态校验后）。

2. 后台控制器权限实现变更
- 变更前：部分接口使用 controller 注解权限。
- 变更后：统一改为 auth 路由授权服务判定。
- 路由与请求结构不变。

3. 超级管理员行为收口
- 新增：系统内置 `super_admin` 角色（不可变更），超管账号强制绑定。
- 新增：超管权限会话兜底为 `menuPermissions=["*"]`，避免因角色配置误操作导致系统不可登录。

4. 权限扩展能力升级
- 新增：角色权限支持自定义权限 key（格式 `/[a-z0-9/_-]+`）。
- 新增：后台/共享写路由支持按首段自动推断权限 key，新增模块无需先改 auth 代码。

5. 后台登录协议升级
- 变更前：`/admin/auth/login` 使用 openId 类标识登录。
- 变更后：`/admin/auth/login` 使用 `loginName + password`。
- 影响：Admin 前端登录页与后台用户创建页已完成联动改造。

## 4. 本次行为更新（2026-03-05）

1. 角色管理控制层瘦身
- 变更前：`AdminRoleController` 内包含角色创建/更新/启停/删除完整业务编排。
- 变更后：业务编排统一下沉到 `AdminRoleManageService`，控制层只保留协议转发。

2. 后台会话归一化逻辑下沉
- 变更前：`AdminAuthController` 自行处理“后台接口收到小程序身份时转游客态”的逻辑。
- 变更后：逻辑统一下沉到 `AuthPortalService.adminValidate/adminMe`。

3. 权限自动化规则明确化
- 路由鉴权和菜单码返回统一来源为 `AuthRoutePermissionResolver`。
- 新增模块默认走“路由自动推断”；仅在不一致场景使用 `@AdminMenuCode`。

## 5. 前端联调检查项

1. 全部认证入口必须使用：
- 后台：`/api/v2/admin/auth/*`
- 小程序：`/api/v2/miniapp/auth/*`

2. 上传接口必须带 token
- `POST /api/v2/files/upload` 不再接受匿名调用。

3. 菜单权限缺失时的预期
- 后台用户调用受控接口会返回统一未授权业务码（HTTP 200 + 业务错误码）。
