# `amap` API 迁移清单（最终版，2026-03-06）

## 1. 迁移目标
1. 后台与小程序接口严格分离。
2. 下线旧共享路径 `/api/v2/amap/**`。
3. 额度统计改为“真实高德远程调用次数”，缓存命中不计费。
4. 审计支持按 `weather/location` 与缓存来源监控。

## 2. 后端迁移结果（DaHeServeV2）

### 2.1 路由分离
1. 小程序：`/api/v2/miniapp/amap/**`
2. 后台：`/api/v2/admin/amap/**`
3. 旧共享：`/api/v2/amap/**` 已下线

### 2.2 新增/增强后台接口
1. `GET /api/v2/admin/amap/regions/provinces`
2. `GET /api/v2/admin/amap/regions/cities`
3. `GET /api/v2/admin/amap/regions/districts`
4. `GET /api/v2/admin/amap/regions/townships`
5. `GET /api/v2/admin/amap/audits/overview`
6. `POST /api/v2/admin/amap/audits/purge`

### 2.3 配额与缓存策略接口增强
1. `PUT /api/v2/admin/amap/quota` 现支持配置：
- Redis缓存开关与前缀
- 区划/天气 TTL
- 本地缓存容量
- 审计自动清理与保留天数

### 2.4 计费口径迁移
1. 旧口径：在应用层按请求次数估算计费。
2. 新口径：在 `AmapOpenServiceImpl#doGetJson` 按真实高德远程调用计费。
3. 分类口径：统一为 `weather`、`location`。

### 2.5 审计来源迁移
1. `backend-proxy`
2. `backend-proxy-cache:{source}`
3. `backend-proxy-mixed:{source...}`

## 3. 前端迁移结果（DaHeAdminV2）
1. `AmapAudit.vue`：
- 新增配置折叠能力（默认收起）。
- 调用日志区域作为主视图。
- 新增 `apiType/requestSource` 过滤。
- 对接概览接口 `/admin/amap/audits/overview`。
- 对接清理接口 `/admin/amap/audits/purge`。

2. `FieldManage.vue`：
- 区划接口迁移为 `/admin/amap/regions/*`。

## 4. 小程序对接状态（DaHeAppV2）
1. 小程序端通过 `/miniapp/amap/*` 访问后端代理能力。
2. 小程序不配置高德 key，由后端统一代理与缓存。
